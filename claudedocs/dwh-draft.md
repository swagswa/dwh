# DWH - Черновик архитектуры

## Что строим

Корпоративное хранилище знаний (DWH) с семантическим поиском и RAG-системой.

Цели:
- Лиды от консалтинга
- Новые бизнес-проекты

Стек: Supabase (PostgreSQL + pgvector), OpenAI text-embedding-3-small
Пользователь системы: клиент лично (1 человек)
Язык данных: в основном русский, немного английского


## Этап 1 - Выгрузка данных + Supabase + лёгкий UI

### Источники данных

ChatGPT Business (ВЫЯСНЕНО)
- 1 пользователь
- ~500 чатов, 1-2 новых в неделю
- Основной паттерн - дописывание в старые чаты (нужно отслеживать обновления, не только новые)
- Нет встроенного экспорта, нужен скрейпинг
- Референс: https://github.com/ryanschiang/chatgpt-export (консольный JS-скрипт, ручной)
- Нужна автоматизация: Playwright или браузерное расширение
- Риск: бан аккаунта при агрессивном скрейпинге

Gmail (ВЫЯСНЕНО)
- 1 ящик
- Глубина: 1 год
- Вложения: уточнить (скорее всего текст писем достаточно)
- Техническое решение: Gmail API (OAuth2) - официальный, безопасный, бесплатный

Telegram каналы (ВЫЯСНЕНО)
- 2 своих канала (есть админ-доступ)
- Менее 100 сообщений всего - маленький объём
- Техническое решение: Bot API (свои каналы, админ-доступ) - проще и стабильнее чем userbot

Сайты (ВЫЯСНЕНО)
- 2 сайта + 2 лендинга = 4 страницы
- Просто веб, без авторизации
- Будут добавляться новые со временем
- Техническое решение: простой HTTP парсинг (cheerio/beautifulsoup), без Playwright

Оффлайн документы (ВЫЯСНЕНО)
- Разные форматы (PDF, Word, Excel и т.д.)
- Мало документов
- Почти всё уже загружено в GPT (дублирование с ChatGPT данными)
- Будут добавляться новые
- Техническое решение: pypdf, python-docx, openpyxl для извлечения текста

### Supabase - структура БД (предварительно)

Таблица documents:
- id (bigserial PK)
- source (text) - chatgpt/gmail/telegram/sites/offline
- source_id (text) - уникальный ID из источника (chat_id, message_id, url и т.д.)
- title (text)
- content (text)
- metadata (jsonb) - специфичные поля по источнику
- embedding (vector 1536)
- created_at (timestamptz)
- updated_at (timestamptz)
- content_hash (text) - для отслеживания изменений в чатах ChatGPT

Таблица sync_log:
- id (bigserial PK)
- source (text)
- source_id (text)
- last_synced_at (timestamptz)
- status (text) - success/error
- error_message (text)
